# Archive Console (local operator panel)

**Archive Console** is a **local operator panel** for **yt-dlp**-oriented **monthly batch** workflows (`monthly_*.bat`), shared **`yt-dlp.conf`**, cookies, and **`report.html`** runs—not a generic download manager. **Not affiliated with yt-dlp**; you provide yt-dlp installs, repo scripts, and config.

Small **FastAPI + SSE** UI that runs the three **PRIMARY** monthly batch drivers with **live logs**, browses **allowlisted** archive folders, and opens **`report.html`** safely. Binds to **127.0.0.1 only** — not exposed on the LAN.

## Where is the server?

The **browser tab is not the server** — it is only an HTTP client. **Closing the tab does not stop uvicorn.**

| What you see | Where uvicorn runs |
|----------------|-------------------|
| Default **`start_archive_console.bat`** | A **separate** console titled **`Archive Console (uvicorn)`** on the taskbar. The Cursor/IDE terminal usually **returns immediately** after launching it. |
| **`ARCHIVE_CONSOLE_ATTACHED=1`** | Uvicorn runs **in the current terminal** (useful from Cursor); **Ctrl+C** stops it. |
| **`start_archive_console_tray.bat`** | Tray runs **`archive_console\tray_app.py`** under **`.venv\Scripts\pythonw.exe`** (detached via `start` so the launcher **`cmd`** exits). **Spawn mode:** the tray **parent** starts a **child** uvicorn process; a watchdog exits the tray when that child dies (e.g. **Settings → Stop**), so the icon usually does not orphan. **Attach mode** (tray started while the port already answers health): only **Exit** closes the tray; **Settings → Stop** stops uvicorn but leaves the tray until **Exit**. Use **`stop_server.ps1`** when you need to kill from a script. |

**Why Explorer used to show a console for the tray:** the old launcher ran **`python.exe tray_app.py`** in the same **`cmd`** session as the **`.bat`**, so the window stayed open for the whole tray lifetime. The current launcher uses **`pythonw.exe`** (no GUI console) and **`start "" /D …`** so the batch can **exit** after kicking off the tray. **`start_archive_console.bat`** is unchanged: it still uses **`python.exe`** and **`_launch_uvicorn.bat`** (or attached mode) for **visible** logging.

- **Port in use but `/api/health` fails:** the launcher detects a listener on your configured port (from **`state.json`**, default **8756**) and offers **Y/N** to kill **only** matching **Python** processes whose command line includes **`uvicorn`** and **`app.main:app`**. Set **`ARCHIVE_CONSOLE_REPLACE=1`** to skip the prompt and kill automatically. Narrow kill logic: **`archive_console/stop_server.ps1`** (never `Stop-Process -Name python`).
- **Inspect a port:** `Get-NetTCPConnection -State Listen -LocalPort 8756 | Select-Object OwningProcess, LocalAddress` (change the port if needed).
- **Manual stop (same rules as the launcher):** `powershell -NoProfile -ExecutionPolicy Bypass -File archive_console\stop_server.ps1` — add **`-Force`** for non-interactive kill. **`-ListOnly`** prints candidate PIDs only.

## P0.5 — Inputs & config editor

- **Nav:** **Inputs & config** — tabs for `playlists_input.txt`, `channels_input.txt`, `videos_input.txt`, `yt-dlp.conf`, and **`cookies.txt`**. **`gallery-dl.conf`** is edited from the dedicated **gallery-dl.conf** sidebar item (same `GET`/`PUT` API as other allowlisted files).
- **Download output folders:** optional **relative** roots (under **archive root**) per job — saved in **`state.json`** as **`download_dirs`** — including **One-off** (`oneoff`, default **`oneoff/`**). Paths must match **Settings → allowlisted paths** (same policy as **Library**). On each **Run**, the server sets **`ARCHIVE_OUT_PLAYLIST`**, **`ARCHIVE_OUT_CHANNEL`**, or **`ARCHIVE_OUT_VIDEOS`** before spawning the monthly `.bat`; **One-off** uses **`ARCHIVE_OUT_ONEOFF`** (from **`download_dirs.oneoff`** or overridden per start). The Python drivers use those when set, otherwise **`playlists/`**, **`channels/`**, **`videos/`**, **`oneoff/`** as today. **…** per field calls **`POST /api/settings/download-dirs/browse`**, which runs a **tkinter** folder dialog on the **Archive Console host** (Windows desktop recommended; headless/Linux may get **503** — type paths manually). **`POST /api/settings/download-dirs/preview`** updates the effective-path panel without saving. Picking the archive root itself is rejected. This does not change `cookies.txt` or `yt-dlp.conf` locations.
- **Read/Write API:** `GET` / `PUT` **`/api/files/<filename>`** where `<filename>` is exactly one of those basenames (no paths). **Never invokes yt-dlp** on save.
- **cookies.txt:** default **locked** (no content in the UI until **Unlock cookies** + warning). `PUT` requires `unlock_cookies: true` in the JSON body.
- **Backups:** before overwrite, the server copies the prior file to **`archive_console/backups/<filename>.<UTC>.bak`** and rotates (count from **`editor_backup_max`** in `state.json`, default **10**). The folder is **gitignored**.
- **Soft checks:** optional **strip blank lines on save** (the three `*_input.txt` lists only); optional **conf smoke** (heuristic hints for `yt-dlp.conf`, non-fatal).
- **Concurrency:** **`PUT` is blocked (409)** while a **Run** tab job is **running** so you are not saving over files a batch job may read (`yt-dlp.conf`, **`gallery-dl.conf`**, `cookies.txt`, input lists). `GET` remains allowed (read-only inspection is still racy but often useful).

## Backend config files (disk = source of truth)

Allowlisted basenames under **archive root** only (see `app/editor_files.py` / `GET`/`PUT` **`/api/files/<filename>`**). One **primary** UI surface per backend config file the app actually passes to a tool.

| Integrated tool | On-disk file (typical) | Passed to subprocess | Primary UI |
|-----------------|------------------------|----------------------|------------|
| **yt-dlp** (monthly bats, One-off, optional gallery fallback) | `yt-dlp.conf` | yt-dlp reads default config from cwd | **yt-dlp.conf** nav (interactive) + raw tab under **Inputs & config** |
| **gallery-dl** (Galleries tab, preview API) | `gallery-dl.conf` (optional) | `-c` when file exists | **gallery-dl.conf** nav (raw JSON editor) |
| **Netscape cookies** (shared) | `cookies.txt` | `--cookies` when present | **Inputs & config → cookies.txt** only (locked by default) |

**Not a project config file:** **Library → clip export** runs **ffmpeg** with **CLI arguments built in code** (`app/clip_export.py` — `build_ffmpeg_argv`). There is **no** on-disk `ffmpeg.conf` unless a future change passes one into ffmpeg; encoding defaults live in code / per-export UI (and optional **Settings → ffmpeg_exe** only selects the binary).

**Not integrated:** **streamlink** — no console UI or repo-managed streamlink config.

When adding a new CLI backend, add the basename to **`EDITABLE_FILENAMES`**, wire the driver to read that path from archive root, add a dedicated nav or clearly labeled editor, and document upstream.

## yt-dlp.conf (interactive editor)

**Nav:** **yt-dlp.conf** (subtitle *Interactive editor*) — not the generic **Settings** page. Tier A controls (sliders, toggles, selects), built-in **presets**, **User preferences** snapshot, Tier B extra flags, live **command preview**, and a **serialized conf** mirror for the **same `yt-dlp.conf`** file the monthly `.bat` drivers use.

**Primary nav (label, subtitle):** Run — *Monthly drivers*; One-off — *Single URL*; Galleries — *gallery-dl*; History & reports — *Ledger & artifacts*; Library — *Browse & clip*; **Rename** — *DeepL & preview*; Inputs & config — *Lists & raw files*; **yt-dlp.conf** — *Interactive editor*; **gallery-dl.conf** — *Gallery-dl config*; **Supported sites** — *yt-dlp & gallery-dl*; Settings — *Console & scheduler*.

- **Supported sites:** read-only **one-pager** fed by **`GET /api/supported-sites`** (optional **`?refresh=true`** to bypass a short in-memory cache). The server runs your local **`yt-dlp --list-extractors`** and **`gallery-dl --list-extractors`** in a worker thread with **timeouts** and **stdout size caps**; links point to **official** project docs only. If **gallery-dl** is missing from **PATH**, the gallery-dl section explains the gap (no 500). Not affiliated with either upstream project.

## Rename pipeline (DeepL + ExifTool metadata)

- **Nav:** **Rename** — queue allowlisted files (from **Library** → **Queue for rename**), choose **Pipeline** (DeepL and/or **ExifTool** template, step order), **Refresh preview**, then **Apply**. **History & reports → Rename log** stores `rename_runs` in **`state.json`** (no API keys).
- **ExifTool (read-only):** Server runs **`exiftool -j -n`** per file (subprocess, argument list, no shell). **Settings → General:** **ExifTool executable** (blank = `exiftool` on PATH) and **timeout per file**. Preview shows key tags / resolved token summary; failures surface per-row with stderr tail (truncated). **Apply** uses **`Path.rename`** with names computed in Python — ExifTool is not used to write files.
- **Template tokens (v1):** `{Stem}` (stem after prior pipeline steps), `{DateTimeOriginal:FORMAT}` / `{CreateDate:FORMAT}` / `{MediaCreateDate:FORMAT}` (first date found in a fallback chain across EXIF / QuickTime / `CreateDate` / `MediaCreateDate` / `FileModifyDate`), `{Model}`, `{LensModel}`, `{ImageSize}` (normalized `WxH`), or any other tag name present in the JSON. `FORMAT` is Python **`strftime`** (e.g. `%Y%m%d`). Missing-tag policy: **keep stem** (warning) or **skip row** (error).
- **Composition:** `options.pipeline_order` is **`exif_then_deepl`** or **`deepl_then_exif`**. Steps fold in `build_rename_preview` (`app/rename_pipeline.py`); future rules (regex, counter) extend the same ordered list without changing **`preview_id` → apply**.
- **Settings → DeepL:** API key (masked; env **`ARCHIVE_CONSOLE_DEEPL_API_KEY`** overrides at runtime), endpoint **auto** (`:fx` → Free host) or **free** / **pro**, source/target languages.
- **API**

| Method | Path | Role |
|--------|------|------|
| `POST` | `/api/rename/preview` | Body: `rels`, `options` (see `RenamePreviewOptions`: `use_deepl`, `use_exif`, `pipeline_order`, `exif_template`, `exif_missing_policy`, DeepL segment flags), `max_files` — rows include `tags_preview`, `operation`, optional DeepL `usage`, `preview_id` |
| `POST` | `/api/rename/apply` | Body: `preview_id` — re-validates, `Path.rename` per file, appends **Rename log** (`operation` from preview token) |
| `GET` | `/api/rename/history` | Ledger entries (no secrets) |

### Single source of truth

- The canonical file on disk remains **`yt-dlp.conf`** next to the monthly `.bat` files (repo root). The UI **parses** that file into an internal model (`YtdlpUiModel` in `app/yt_dlp_config_model.py`) and **serializes** it back on **Save** with a deterministic **managed** block and a banner comment: `# Generated by Archive Console — last saved <iso> — preset: <id>`.
- **Comments, blank lines, and lines the parser does not map** (unknown options, blocked Tier B keys, malformed values) accumulate in **`preserved_tail`** and are **re-appended verbatim** after the managed block on save. That is how **commented “notes”** and uncommon flags coexist with the UI without being dropped—as long as they are not duplicated as both Tier A fields and raw lines (prefer Tier B for extra flags you want round-tripped in the managed section).
- **Tier A** is a curated set of safe, typed fields (~40–80 over time). **Tier B** (`extra_kv`) is a searchable key/value list for other hyphenated options; keys are validated (no cookies, no shell metacharacters, no newlines). **Tier C** is the read-only **serialized output** panel; full freeform editing stays under **Inputs & config → `yt-dlp.conf`**. If you edit raw elsewhere, **Recapture User preferences** or reload the page to resync from disk.
- **`archive_console/yt_dlp_ui_state.json`** (gitignored; see **`yt_dlp_ui_state.example.json`**) stores **`active_preset_id`**, **`user_preferences_snapshot`** (fields captured from disk), and optional custom preset metadata — **not** cookie secrets.

### Presets

- **Built-ins** (e.g. Balanced, Best quality, Fast / low bandwidth, Audio only, Merge-friendly, Debug verbose) **replace Tier A fields** from a documented baseline + overlay (`app/yt_dlp_presets.py`); **`extra_kv`** and **`preserved_tail`** are not cleared by **Apply** unless you save after applying.
- **User preferences:** use **Capture / Recapture** to store the **parsed model** (and optional snapshot) from the current **`yt-dlp.conf`**. Applying this preset restores that snapshot into the editor; **Save** writes it to disk.
- Presets do **not** embed **Google account cookies** or raw `cookies.txt` content.

### API (127.0.0.1 only; same host allowlist as the rest of the app)

| Method | Path | Role |
|--------|------|------|
| `GET` | `/api/ytdlp/setup` | Load model from `yt-dlp.conf`, UI state, preset list, Tier A schema |
| `POST` | `/api/ytdlp/setup/preview` | Return `yt-dlp` CLI preview string (no subprocess) |
| `POST` | `/api/ytdlp/setup/save` | Backup + write `yt-dlp.conf`, `config_smoke`; **409** if a run is active |
| `POST` | `/api/ytdlp/setup/apply-preset` | Merge built-in preset into current model |
| `POST` | `/api/ytdlp/setup/capture-user` | Re-read disk into **User preferences** snapshot |

### Expanding Tier A over time

Add a field to **`YtdlpUiModel`**, **`SERIALIZE_ORDER`** in `app/yt_dlp_conf_io.py`, and a control definition in **`TIER_A_GROUPS`** in `app/yt_dlp_config_model.py`. Re-run **`python -m pytest`**.

## P0 (shipped)

- **Start:** double-click **`start_archive_console.bat`** in the repo root. It creates **`archive_console\.venv`**, installs **`requirements.txt`**, and either **opens the browser** if **`/api/health`** already passes, or starts **uvicorn** in a titled console **`Archive Console (uvicorn)`** (or attached mode / tray — see **Where is the server?** above). Stale listeners on the same port trigger a **safe kill** prompt or **`ARCHIVE_CONSOLE_REPLACE=1`**.
- **URL:** `http://127.0.0.1:<port>/` (default port **8756**, overridable in **`archive_console/state.json`** after first save from **Settings** — restart required for port changes).
- **Stop server:** **Settings → Danger zone → Stop Archive Console server** — you must type **`SHUTDOWN`** to confirm. Calls **`POST /api/shutdown`** with JSON **`{"confirm":"SHUTDOWN"}`** (never **GET**). Requests are accepted only from **loopback** clients (`127.0.0.1` / `::1`). Set optional env **`ARCHIVE_SHUTDOWN_TOKEN`** to require header **`X-Archive-Shutdown-Token`** (same value); use that on multi-user machines where anyone with local browser access could otherwise open devtools. The handler schedules **`app.shutdown.request_shutdown`** (deferred **`os._exit(0)`**) so the JSON response can finish first — **lifespan `finally` hooks may not fully run**. The web UI then replaces the document with a static **“Server stopped”** page (avoids broken overlays after the socket drops). **Tray spawn mode:** when the uvicorn **child** exits, the tray **watchdog** calls **`icon.stop()`** so the tray icon does not stay behind. **Tray Exit** in **spawn** mode still tries HTTP shutdown first, then **`terminate`** / **`taskkill`** if the child does not exit. **Attach** mode: tray **Exit** does **not** stop uvicorn; **Settings → Stop** frees the port but the tray icon remains until **Exit**.
- **Runs:** **Run** tab spawns **`monthly_watch_later_archive.bat`**, **`monthly_channels_archive.bat`**, **`monthly_videos_archive.bat`** with **`ARCHIVE_CONSOLE_UNATTENDED=1`** so interactive **`pause`** steps are skipped (double-clicking the bats alone is unchanged unless that env var is set). **Cookie preflight (default on):** the server does **not** start the batch until **`POST /api/run/start`** includes **`cookie_confirm: true`** after the UI modal, or returns **HTTP 428** with **`error: "cookie_confirm_required"`**. This is skipped for **`dry_run: true`** and for **scheduled** in-process runs (only the Run tab gate). Live log uses **full main column width**, horizontal scroll by default (optional **Wrap long lines**), optional **Highlight output** (browser-only severity styling + **Last progress** hint from `[download] …%` lines; preference **`localStorage`** key **`archive_console_log_highlight`** = `1` / unset), monospace sizing **A− / A+**, and **Stop run** (with confirm) calls **`POST /api/run/stop`**, which ends **only** the **cmd.exe** process tree the server started for that job (Windows: **`taskkill /PID … /T /F`** on that tracked root PID — not a blanket **`python`** kill). **Stopping mid-run can leave partial state** (e.g. download-archive / temp files / incomplete merges — same general orphan risk as **Ctrl+C** or killing the batch window). The log shows a **stopped** line; exit is recorded as **canceled** in history; use **History & reports** (or **`logs/latest_run*.txt`**) after a clean run when in doubt. Subprocess output is **piped** (not a TTY); **disk** logs under **`logs/archive_run_*`** stay plain — highlighting applies only in the UI.
- **Env toggles in UI:** **`ARCHIVE_DRY_RUN=1`**, **`SKIP_PIP_UPDATE=1`**, **`SKIP_YTDLP_UPDATE=1`** — same semantics as documented in **`BAT_FILES.md`** / **`README.md`**. Dry-run also skips the cookie preflight gate.
- **One-off:** **One-off** tab runs **`archive_oneoff_run.py`** with **`python -u`** using the **driver interpreter**: **`{archive_root}/archive_console/.venv/Scripts/python.exe`** (Windows) when that venv exists (same tree as **`start_archive_console.bat`**), otherwise **`{archive_root}/.venv/…`**, else the uvicorn interpreter. **`archive_console/requirements.txt`** includes **`yt-dlp[default]`** so that venv can **`import yt_dlp`** after **`pip install -r requirements.txt`**. Cwd is **archive root**; env vars **`ARCHIVE_ONEOFF_URL`**, **`ARCHIVE_OUT_ONEOFF`**, etc. — **only one** job (monthly, **One-off**, or **Galleries**) may run at a time (**409** if busy). Cookie **HTTP 428** + modal matches **Run** (skipped for dry-run). Download archive file **`oneoff_downloaded.txt`** (does not share **`videos_downloaded.txt`**). Per-run artifacts still land under **`logs/archive_run_<UTC>/`**; pointer **`logs/latest_run_oneoff.txt`**. Rolling summary **`logs/oneoff_report/summary.jsonl`** + **`report.html`**; epoch length from **`oneoff_report_retention_days`** in **`state.json`** (saved with **Settings → Storage & retention → Save retention prefs**). While the **One-off** view is visible, a **45-minute** dismissible cookie hygiene banner may appear; **Acknowledge** posts **`POST /api/settings`** with **`oneoff_cookie_reminder_last_unix`** (same persisted field as Settings); an optional alias is **`POST /api/oneoff/cookie-reminder-ack`**.
- **Galleries:** **Galleries** tab runs **`archive_gallery_run.py`** (same **single-job mutex** as monthly / One-off). **`gallery-dl`** must be installed separately and on **PATH** (or set per-start **`gallery_dl_exe`** in the API body — UI uses defaults). **Preview** calls **`POST /api/galleries/preview`** → server runs **`gallery-dl -s -j`** (no media files; still hits the network). **Run** sets **`ARCHIVE_OUT_GALLERIES`** from **`download_dirs.galleries`** (default folder **`galleries/`**) or a per-run allowlisted override; optional **`gallery-dl.conf`** at archive root (primary editor: **gallery-dl.conf** in the sidebar — JSON per [gallery-dl configuration](https://github.com/mikf/gallery-dl/blob/master/docs/configuration.rst)); **`cookies.txt`** is passed when present (edit under **Inputs & config**). The checked-in **`gallery-dl.conf`** is written for **gallery-dl 1.31.x**: **`base-directory` `.`** so each run’s **`-o …/gallery_<stamp>/`** stays the root; **`directory`** **`{category}`** / **`{subcategory}`** for layout; **`path-restrict`:** **`windows`**; modest **timeout**, **retry**, and **429** backoff (no cookie blobs — use **`cookies.txt`** only). Artifacts: **`logs/archive_run_<UTC>/`** (`summary.txt`, **`report.html`**, **`verification.json`**, **`preview_snapshot.json`** when a preview was sent), pointer **`logs/latest_run_galleries.txt`**. Optional **Video fallback** runs **`python -m yt_dlp`** for **`v.redd.it`** URLs from the preview that still lack files after **gallery-dl**. **`GET /api/health`** includes **`gallery_dl_on_path`** (boolean from **`shutil.which`**).
- **State:** no database; **`archive_console/state.json`** (gitignored) stores settings, P1 placeholders, and **run history**. Copy from **`state.example.json`** if needed.
- **Security:** path **allowlist** under **`archive_root`** (defaults to the parent of **`archive_console`**, i.e. the scripts folder). Traversal segments (`..`) are rejected (**403**).
- **Serving artifacts**
  - **`GET /reports/file?rel=…`:** **`Content-Type`** follows the suffix (`.html`, `.mp4`, `.webm`, `.mkv`, `.webp`, …). Default is **`Content-Disposition: inline`** with a filename hint so browsers **render or play in-tab** instead of forcing a download. **`?download=1`** or **`disposition=attachment`** sets **`attachment`**. **`cookies.txt`** is **never** served here (**403**).
  - **`GET /reports/view?rel=…/report.html`:** returns the same HTML after **rewriting `file://` `href` / `src`** that point under **`archive_root`** to same-origin **`/reports/file?rel=…`**, and injects a small script so **JavaScript-built** filepath links (still `file:` at generation time) navigate via **`/reports/file`** when the page is loaded over **`http://127.0.0.1`**. **Why:** a page on `http://127.0.0.1` cannot treat **`file://`** links like double-clicking files; mixed-origin behavior blocks or mishandles them compared to opening the report via **`file:///…`**. Rewriting is intentional so video/image links match **Open** from the console. The shim **no-ops on `file:` protocol** so opening the report from disk is unchanged.
  - **Range requests:** Starlette **`FileResponse`** sets **`Accept-Ranges: bytes`** and supports **`Range` / `206 Partial Content`** for large media when the client sends **`Range`** (essential for seeking in **MP4/WebM** in typical browsers).
- **Library → Player (local queue) + clip export:** The **Library** tab embeds an HTML **`<video controls>`** playlist. The element’s `src` is the same allowlisted **`GET /reports/file?rel=…`** URL used by **Open** (not **`file://`**). Every request is re-checked against the allowlist; **`cookies.txt`** still **403** here. **`GET /api/files/playable-enumerate`** lists playable files **in one folder only** (the `recursive` query is ignored); responses are capped (default **1000**, max **2000**). The UI queue is persisted in **`localStorage`** (`archive_console_library_player_v1`, with one-time migration from `archive_console_files_player_v1`; fields include **`loopPlaylist`** and **`shuffle`**). On **`ended`**, the player auto-advances; **loop** (toggle) wraps the **entire** queue; **shuffle** **on** applies a new **Fisher–Yates** order, **off** restores add order. **Toolbar:** icon buttons with **`aria-pressed`** for shuffle/loop. **Queue:** double-click a row to play now; drag-drop reorders. **List selection (Explorer-like):** Ctrl/⌘+click, Shift+range; **Add folder** uses the first selected folder or current directory. **Control matrix:** **`FILES_PLAYER_AUDIT.md`**. **Firefox** recommended for codec coverage.
  - **Clip export (local files only):** **`POST /api/clip/start`** runs **ffmpeg** on the **Archive Console host** for an allowlisted **source** file (playable media suffixes only) into an **existing** allowlisted **output directory** ( **`POST /api/clip/browse-output`** uses the same native folder picker pattern as download-dir browse). **One clip at a time** (**409** if busy); this path does **not** share the **Run** tab mutex (monthly / One-off / Galleries jobs can still be running — only one of those at a time). **`GET /api/clip/status`** reports phase / stderr tail. Completed clips append **`run_history`** with **`job: "clip_export"`** (History **Run folder** column links the output file via **`/reports/file`**). **Formats:** **MP4** (H.264+AAC), **WebM** (VP9+Opus), **GIF** (ffmpeg **`palettegen` / `paletteuse`**, 15 fps, 480px width — good for short loops; large or long clips get huge files). There is **no** separate **`ffmpeg.conf`** on disk; argv is built in **`app/clip_export.py`**. **`ffmpeg_exe`** in **`state.json`** (blank = **`ffmpeg`** on **PATH**); editable under **Settings → General**.
  - **Media details (MediaInfo):** Selecting a **file** in Library loads **`GET /api/files/mediainfo?rel=…`** asynchronously (does not block listing). The server runs **`mediainfo --Output=JSON`** in a worker thread (timeout, 64 GiB size guard). **`mediainfo_exe`** in **`state.json`** (blank = **`mediainfo`** on **PATH**); **Settings → General**. Clear errors if the CLI is missing or fails.
  - **Duplicate finder (exact files):** **Pure Python** backend in-process: recurse under operator-selected **allowlisted** roots, bucket by **file size**, then **SHA-256** (streaming) within multi-file buckets. **No** perceptual / “similar re-encode” detection in v1. **`POST /api/duplicates/scan`** + poll **`GET /api/duplicates/status`** (**409** if a scan is already running). **`POST /api/duplicates/apply`** with **`dry_run: true`** previews; non–dry-run requires JSON **`confirm: "DELETE_DUPLICATES"`** (the UI also uses confirm + typed phrase). **Quarantine** moves files under **`duplicates_quarantine_rel`** (default **`logs/_duplicates_quarantine`**, must stay allowlisted — set in **Settings → General**); **delete** removes files permanently. **Independent** of the Run-tab mutex (same idea as clip export).
  - **Library height model (layout):** **`main`** is a **flex column** with **`min-height: 100vh`**; **`#view-library`** (when active) uses **`flex: 1; min-height: 0`** so the **card** can use the column below the Library header. The **two panes share one fixed outer box:** **`#filesWorkspaceShell`** has a **single pixel height** (`--files-workspace-height`, set in JS). Folder changes only update **`#fileList`** scroll metrics — **not** this shell. **Default** height uses **almost all space from the shell’s top to the bottom of the viewport** (no separate **92vh** cap on that default); CSS falls back to **`min(88vh, 900px)`** before JS runs. **`localStorage`** key **`library.workspace.height`** persists the operator’s choice (migrates from legacy **`files.workspace.height`**). Clear that key or drag the resize strip if an old saved height still feels short. URL **`?view=files`** is accepted as an alias for **`?view=library`**.
  - **Workspace resize:** A **horizontal strip** under the shell (**`#filesWorkspaceResizeY`**, **`ns-resize`**) drags to grow or shrink **both columns together**; bounds are **~480px** minimum and **viewport bottom** maximum. Window **resize** reclamps saved height.
  - **Player column:** **`.files-player`** uses **`overflow-y: hidden`**; only **`.files-player-queue`** scrolls. **`#filesVideoFrame`** height is computed from pane width and aspect ratio with a cap derived from **measured** non-video chrome (see **`FILES_LAYOUT_AUDIT.md`**).

## P1 (scaffold + Phase 1 UI)

- **Settings → Operator backups:** ZIP under an allowlisted path (default `logs/archive_console_backups`), optional `state.json` + `logs/` tree, retention by file count and optional age. **`POST /api/settings/operator-backup/run`** (blocked while a **Run** job is active). Outcomes stored in **`last_operator_backup`** in **`state.json`**.
- **Settings → Storage & retention:** Manual **Preview** / **Run cleanup** for old **`logs/archive_run_*`** folders (directory mtime) and **`archive_console_backup_*.zip`** files under the operator backup destination (file mtime). Options persist in **`state.json`** as **`storage_retention`** (default **90** days). Only paths under **archive root** that pass the **allowlist** are eligible; **`logs/latest_run*.txt` targets are never deleted**; **no cleanup while a Run job is active** (**409**). **`run_history` in `state.json` is not rewritten**—History links may 404 for pruned folders. Broken pointers are cleared after delete. **No automatic pruning on server start** in v1.
- **Settings → Scheduling:** CRUD saved to **`schedules[]`**. **Day-of-month 1–31** is **clamped** in short months (e.g. 31 → Feb 28/29). **Next-run** hints and firing use **local machine time** (optional **`TZ`** env for an IANA zone). When **`features.scheduler_enabled`** is **true** and the server is restarted, an in-process **APScheduler** tick (30s) starts the same **`.bat`** jobs as **Run** at the scheduled wall time; **overlapping** runs are skipped with a log line if a job is still **running**. **Sleep/wake:** missed ticks are **not** replayed.
- **Cookie hygiene:** optional **N-day** in-app reminder (low-priority nudge, max **14** days in UI/state; not a substitute for the Run preflight when that is enabled) + acknowledge / **short snooze** only (**1 h** / **3 h** in Settings, **15 min** on the Run banner; **server clock** is authoritative; multiple tabs are last-write-wins). Run-page + **`GET /api/settings/reminders`** include the cookie banner and the **pre-run** banner for the **earliest upcoming** enabled schedule. No browser automation; **no** logging of `cookies.txt` contents.
- **Tray pre-run balloons (optional):** when **`features.tray_notify_before_schedule`** is **true**, the console POSTs to **`http://127.0.0.1:<tray_notify_port>/notify`** during the **same** pre-run time window as the web banner (T−N minutes through kickoff, local clock). **`tray_app.py`** listens on **localhost only** and shows a **pystray** balloon. Port defaults to **`state.json` `port` + 101** when **`tray_notify_port`** is **0**; change it in **Settings** if it conflicts. If the tray app is not running, the server logs a warning and may record **`tray_notify_last_failure_*`** in **`state.json`** (see Settings).
- **`features.require_cookie_confirm_manual`:** default **`true`** — Run-tab manual starts require the cookie confirmation modal unless disabled in **Settings** or bypassed with **dry-run** (see **Runs** above).
- **`features.scheduler_enabled`:** **`true`** starts the in-process backend on server boot; **`false`** keeps schedules as saved-only (Task Scheduler / tray still useful to keep the process alive).
- **`features.notifications_stub`**: reserved for Windows toasts (**P2**).
- **Task kill / SIGTERM from other tools:** External **`taskkill`** on unrelated processes is out of scope; use **Stop run** in the UI or **`stop_server.ps1`** for the console server itself, not arbitrary yt-dlp PIDs.

## Coexistence with the rest of the repo

- **`BAT_FILES.md`** remains the **authoritative `.bat` inventory**; the console does not replace Task Scheduler or raw double-click runs.
- **`README.md`** quick-start table still describes the three **`monthly_*`** drivers; the console is an optional launcher with streaming logs.
- **Core Python drivers** (`archive_*_run.py`) are **not** forked; the console spawns the existing **`.bat`** files so **pip / backup / pointer** behavior stays aligned with interactive runs.

## Install & troubleshooting

| Issue | What to do |
|--------|------------|
| Browser opens but page / connection fails | The launcher waits with **`Start-Sleep`** via PowerShell (avoids **`ping`** / **`timeout`** quirks in some terminals). Uvicorn is started through **`archive_console/_launch_uvicorn.bat`** so **`cmd /k`** does not have to nest long quoted paths (that parsing bug can make CMD run **`python.exe` as if it were a `.py` file** → `SyntaxError` / `MZ` / `\x90`). Check the **“Archive Console (uvicorn)”** window; ensure **`http://127.0.0.1:<port>/`** matches **`state.json`** `port` (default **8756**). |
| Port already in use | Change **`port`** in **`state.json`** (or stop the other process). The launcher treats “health check succeeds” as “server already up” and only opens the browser. |
| Firewall | **N/A** for `127.0.0.1`-only binding. |
| `python` not found | Install **Python 3.10+** and ensure it is on **PATH** when running the launcher. |
| Blank logs / 409 on Run | Only **one** run at a time; wait for the current job to finish. |
| Reports 403 | Path must fall under **`allowlisted_rel_prefixes`** (default: `logs`, `playlists`, `channels`, `videos`). |
| Bookmark **`/logs`** or **`/reports`** | Redirects to **`/?view=history`** with the appropriate section anchor (same combined **History & reports** screen). |

  5. **yt-dlp.conf** (nav) → change a slider → **Save** → confirm root **`yt-dlp.conf`** updates; restart console and verify values; with a job running, **Save** should show **409**.

If health already passes, tray **attaches** and **Exit** leaves the server running. A small **HTTP** listener on **`127.0.0.1:<tray_notify_port>`** accepts **`POST /notify`** (JSON **`title`**, **`body`**) for optional pre-run cookie reminders when enabled in Settings.

### Browser tab, in-app sidebar, and editor chrome

- **Where the rail icon “lives”:** The **Archive Console** UI sidebar mark uses **`/static/app-icon-128.png`** (**`srcset`** **`app-icon-256.png` 2×**) in **`templates/index.html`**, generated from **`assets/archive-console-icon-master.png`** via **`assets/build_tray_icon.py`**. Optional **`/static/app-icon.svg`** remains a simplified vector only (not used for sidebar or primary favicon). **Nav pills** (**Run**, **Files**, …) are plain buttons with **`.nav-item.is-active`** — gold outline is theme CSS, not a separate icon asset.
- **Favicon / install metadata:** **`index.html`** lists **`favicon.ico`**, **`favicon-32.png`**, **`favicon-16.png`**, and **`app-icon-64.png`** (raster-first so the **master-derived** art shows in tabs). **`manifest.json`** uses **`icon-192.png`** / **`icon-512.png`** only, plus **`theme-color`**.
- **Cursor / VS Code:** Tabs and the **far-left activity bar** are **editor-controlled**. A **generic globe** there is **not** coming from this repo; fixing it is **favicon + reload** for preview tabs, not a change under **`archive_console/static`**. For workspace/window branding, see current **Cursor** / **VS Code** documentation (product features change independently of this app).

### Tray icon (assets)

- **Shipped files:** **`archive_console/assets/tray.ico`** (embedded **16, 24, 32, 48**, plus **256** for sharp rescale) and PNG sources **`assets/tray.png`** (256) and **`assets/tray_64.png`**. Motif: **single gold energy arrow** through a **void tile** with a **vertical light gash** (no folder). Optional **reference render** for art tweaks: **`assets/archive-console-icon-master.png`**. **`build_tray_icon.py`** also writes **`static/favicon.ico`**, **`favicon-16/32.png`**, **`apple-touch-icon.png`**, **`icon-192/512.png`** for tabs / manifest. The tray loads the **largest** ICO frame for pystray, which re-serializes for the shell.
- **Regenerate (maintainers):** from **`archive_console`**, run **`python assets/build_tray_icon.py`**. If **`assets/archive-console-icon-master.png`** exists, outputs are **rescaled** from it via **`icon_master_derivatives.py`** (stepwise **LANCZOS**, mild contrast / unsharp at **≤64 px**). Otherwise **`tray_icon_raster.draw_tray_icon`** fills all sizes. **`static/app-icon-{512,256,128,64,32}.png`** are the explicit derivative family; **`icon-512.png`**, favicons, and **`tray.ico`** are regenerated from the same pipeline.
- **Dev vs frozen path:** runtime resolves **`archive_console/assets/tray.ico`**. Under **PyInstaller** (Windows), bundle the ICO next to the extract root as **`assets/tray.ico`**, for example from **`archive_console`**:  
  **`pyinstaller --noconsole --onefile --hidden-import tray_icon_raster --add-data "assets/tray.ico;assets" tray_app.py`**  
  (Use **`;`** as the **`--add-data`** separator on Windows; on Linux/macOS use **`:`.**)
- **Missing file:** the app logs a **warning** and draws a **generated** black/gold chip (same geometry as the builder) so behavior never silently falls back to an unrelated default.
- **HiDPI:** **`tray_app`** attempts **per-monitor DPI awareness** on Windows before creating the icon (best effort). If the icon looks soft at **125% / 150%**, confirm the process is not **system DPI–unaware** (Python / frozen manifest) and that **`tray.ico`** contains crisp **16–32** entries.
- **Stale icon in the shell (rare):** Windows may cache notification-area images. If an old tray icon persists after an update, try **sign out/in**, restarting **Explorer**, or (advanced): **`ie4uinit.exe -show`** — see Microsoft / community notes on notification icon refresh.
