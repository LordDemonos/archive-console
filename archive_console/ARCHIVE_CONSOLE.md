# Archive Console (local operator panel)

**Archive Console** is a **local operator panel** for **yt-dlp**-oriented **monthly batch** workflows (`monthly_*.bat`), shared **`yt-dlp.conf`**, cookies, and **`report.html`** runs—not a generic download manager. **Not affiliated with yt-dlp**; you provide yt-dlp installs, repo scripts, and config.

Small **FastAPI + SSE** UI that runs the three **PRIMARY** monthly batch drivers with **live logs**, browses **allowlisted** archive folders, and opens **`report.html`** safely. Binds to **127.0.0.1 only** — not exposed on the LAN.

## Where is the server?

The **browser tab is not the server** — it is only an HTTP client. **Closing the tab does not stop uvicorn.**

| What you see | Where uvicorn runs |
|----------------|-------------------|
| Default **`start_archive_console.bat`** | A **separate** console titled **`Archive Console (uvicorn)`** on the taskbar. The Cursor/IDE terminal usually **returns immediately** after launching it. |
| **`ARCHIVE_CONSOLE_ATTACHED=1`** | Uvicorn runs **in the current terminal** (useful from Cursor); **Ctrl+C** stops it. |
| **`start_archive_console_tray.bat`** | Uvicorn runs **headless** from a **tray** process; use **Quit** in the menu or **`stop_server.ps1`**. |

- **Port in use but `/api/health` fails:** the launcher detects a listener on your configured port (from **`state.json`**, default **8756**) and offers **Y/N** to kill **only** matching **Python** processes whose command line includes **`uvicorn`** and **`app.main:app`**. Set **`ARCHIVE_CONSOLE_REPLACE=1`** to skip the prompt and kill automatically. Narrow kill logic: **`archive_console/stop_server.ps1`** (never `Stop-Process -Name python`).
- **Inspect a port:** `Get-NetTCPConnection -State Listen -LocalPort 8756 | Select-Object OwningProcess, LocalAddress` (change the port if needed).
- **Manual stop (same rules as the launcher):** `powershell -NoProfile -ExecutionPolicy Bypass -File archive_console\stop_server.ps1` — add **`-Force`** for non-interactive kill. **`-ListOnly`** prints candidate PIDs only.

## P0.5 — Inputs & config editor

- **Nav:** **Inputs & config** — tabs for `playlists_input.txt`, `channels_input.txt`, `videos_input.txt`, `yt-dlp.conf`, and **`cookies.txt`**.
- **Download output folders:** optional **relative** roots (under **archive root**) per job — saved in **`state.json`** as **`download_dirs`**. Paths must match **Settings → allowlisted paths** (same policy as **Files**). On each **Run**, the server sets **`ARCHIVE_OUT_PLAYLIST`**, **`ARCHIVE_OUT_CHANNEL`**, or **`ARCHIVE_OUT_VIDEOS`** before spawning the monthly `.bat`; the Python drivers use those when set, otherwise **`playlists/`**, **`channels/`**, **`videos/`** as today. **…** per field calls **`POST /api/settings/download-dirs/browse`**, which runs a **tkinter** folder dialog on the **Archive Console host** (Windows desktop recommended; headless/Linux may get **503** — type paths manually). **`POST /api/settings/download-dirs/preview`** updates the effective-path panel without saving. Picking the archive root itself is rejected. This does not change `cookies.txt` or `yt-dlp.conf` locations.
- **Read/Write API:** `GET` / `PUT` **`/api/files/<filename>`** where `<filename>` is exactly one of those basenames (no paths). **Never invokes yt-dlp** on save.
- **cookies.txt:** default **locked** (no content in the UI until **Unlock cookies** + warning). `PUT` requires `unlock_cookies: true` in the JSON body.
- **Backups:** before overwrite, the server copies the prior file to **`archive_console/backups/<filename>.<UTC>.bak`** and rotates (count from **`editor_backup_max`** in `state.json`, default **10**). The folder is **gitignored**.
- **Soft checks:** optional **strip blank lines on save** (the three `*_input.txt` lists only); optional **conf smoke** (heuristic hints for `yt-dlp.conf`, non-fatal).
- **Concurrency:** **`PUT` is blocked (409)** while a **Run** tab job is **running** so you are not saving over files yt-dlp may be reading. `GET` remains allowed (read-only inspection is still racy but often useful).

## Download settings (interactive yt-dlp)

**Nav:** **Download settings** — Tier A controls (sliders, toggles, selects), built-in **presets**, **User preferences** snapshot, Tier B extra flags, live **command preview**, and a **serialized conf** mirror.

### Single source of truth

- The canonical file on disk remains **`yt-dlp.conf`** next to the monthly `.bat` files (repo root). The UI **parses** that file into an internal model (`YtdlpUiModel` in `app/yt_dlp_config_model.py`) and **serializes** it back on **Save** with a deterministic **managed** block and a banner comment: `# Generated by Archive Console — last saved <iso> — preset: <id>`.
- **Comments, blank lines, and lines the parser does not map** (unknown options, blocked Tier B keys, malformed values) accumulate in **`preserved_tail`** and are **re-appended verbatim** after the managed block on save. That is how **commented “notes”** and uncommon flags coexist with the UI without being dropped—as long as they are not duplicated as both Tier A fields and raw lines (prefer Tier B for extra flags you want round-tripped in the managed section).
- **Tier A** is a curated set of safe, typed fields (~40–80 over time). **Tier B** (`extra_kv`) is a searchable key/value list for other hyphenated options; keys are validated (no cookies, no shell metacharacters, no newlines). **Tier C** is the read-only **serialized output** panel; full freeform editing stays under **Inputs & config → `yt-dlp.conf`**. If you edit raw elsewhere, **Recapture User preferences** or reload the page to resync from disk.
- **`archive_console/yt_dlp_ui_state.json`** (gitignored; see **`yt_dlp_ui_state.example.json`**) stores **`active_preset_id`**, **`user_preferences_snapshot`** (fields captured from disk), and optional custom preset metadata — **not** cookie secrets.

### Presets

- **Built-ins** (e.g. Balanced, Best quality, Fast / low bandwidth, Audio only, Merge-friendly, Debug verbose) **replace Tier A fields** from a documented baseline + overlay (`app/yt_dlp_presets.py`); **`extra_kv`** and **`preserved_tail`** are not cleared by **Apply** unless you save after applying.
- **User preferences:** use **Capture / Recapture** to store the **parsed model** (and optional snapshot) from the current **`yt-dlp.conf`**. Applying this preset restores that snapshot into the editor; **Save** writes it to disk.
- Presets do **not** embed **Google account cookies** or raw `cookies.txt` content.

### API (127.0.0.1 only; same host allowlist as the rest of the app)

| Method | Path | Role |
|--------|------|------|
| `GET` | `/api/ytdlp/setup` | Load model from `yt-dlp.conf`, UI state, preset list, Tier A schema |
| `POST` | `/api/ytdlp/setup/preview` | Return `yt-dlp` CLI preview string (no subprocess) |
| `POST` | `/api/ytdlp/setup/save` | Backup + write `yt-dlp.conf`, `config_smoke`; **409** if a run is active |
| `POST` | `/api/ytdlp/setup/apply-preset` | Merge built-in preset into current model |
| `POST` | `/api/ytdlp/setup/capture-user` | Re-read disk into **User preferences** snapshot |

### Expanding Tier A over time

Add a field to **`YtdlpUiModel`**, **`SERIALIZE_ORDER`** in `app/yt_dlp_conf_io.py`, and a control definition in **`TIER_A_GROUPS`** in `app/yt_dlp_config_model.py`. Re-run **`python -m pytest`**.

## P0 (shipped)

- **Start:** double-click **`start_archive_console.bat`** in the repo root. It creates **`archive_console\.venv`**, installs **`requirements.txt`**, and either **opens the browser** if **`/api/health`** already passes, or starts **uvicorn** in a titled console **`Archive Console (uvicorn)`** (or attached mode / tray — see **Where is the server?** above). Stale listeners on the same port trigger a **safe kill** prompt or **`ARCHIVE_CONSOLE_REPLACE=1`**.
- **URL:** `http://127.0.0.1:<port>/` (default port **8756**, overridable in **`archive_console/state.json`** after first save from **Settings** — restart required for port changes).
- **Runs:** **Run** tab spawns **`monthly_watch_later_archive.bat`**, **`monthly_channels_archive.bat`**, **`monthly_videos_archive.bat`** with **`ARCHIVE_CONSOLE_UNATTENDED=1`** so interactive **`pause`** steps are skipped (double-clicking the bats alone is unchanged unless that env var is set). **Cookie preflight (default on):** the server does **not** start the batch until **`POST /api/run/start`** includes **`cookie_confirm: true`** after the UI modal, or returns **HTTP 428** with **`error: "cookie_confirm_required"`**. This is skipped for **`dry_run: true`** and for **scheduled** in-process runs (only the Run tab gate). Live log uses **full main column width**, horizontal scroll by default (optional **Wrap long lines**), optional **Highlight output** (browser-only severity styling + **Last progress** hint from `[download] …%` lines; preference **`localStorage`** key **`archive_console_log_highlight`** = `1` / unset), monospace sizing **A− / A+**, and **Stop run** (with confirm) calls **`POST /api/run/stop`**, which ends **only** the **cmd.exe** process tree the server started for that job (Windows: **`taskkill /PID … /T /F`** on that tracked root PID — not a blanket **`python`** kill). **Stopping mid-run can leave partial state** (e.g. download-archive / temp files / incomplete merges — same general orphan risk as **Ctrl+C** or killing the batch window). The log shows a **stopped** line; exit is recorded as **canceled** in history; use **History & reports** (or **`logs/latest_run*.txt`**) after a clean run when in doubt. Subprocess output is **piped** (not a TTY); **disk** logs under **`logs/archive_run_*`** stay plain — highlighting applies only in the UI.
- **Env toggles in UI:** **`ARCHIVE_DRY_RUN=1`**, **`SKIP_PIP_UPDATE=1`**, **`SKIP_YTDLP_UPDATE=1`** — same semantics as documented in **`BAT_FILES.md`** / **`README.md`**. Dry-run also skips the cookie preflight gate.
- **State:** no database; **`archive_console/state.json`** (gitignored) stores settings, P1 placeholders, and **run history**. Copy from **`state.example.json`** if needed.
- **Security:** path **allowlist** under **`archive_root`** (defaults to the parent of **`archive_console`**, i.e. the scripts folder). Traversal segments (`..`) are rejected (**403**).
- **Serving artifacts**
  - **`GET /reports/file?rel=…`:** **`Content-Type`** follows the suffix (`.html`, `.mp4`, `.webm`, `.mkv`, `.webp`, …). Default is **`Content-Disposition: inline`** with a filename hint so browsers **render or play in-tab** instead of forcing a download. **`?download=1`** or **`disposition=attachment`** sets **`attachment`**. **`cookies.txt`** is **never** served here (**403**).
  - **`GET /reports/view?rel=…/report.html`:** returns the same HTML after **rewriting `file://` `href` / `src`** that point under **`archive_root`** to same-origin **`/reports/file?rel=…`**, and injects a small script so **JavaScript-built** filepath links (still `file:` at generation time) navigate via **`/reports/file`** when the page is loaded over **`http://127.0.0.1`**. **Why:** a page on `http://127.0.0.1` cannot treat **`file://`** links like double-clicking files; mixed-origin behavior blocks or mishandles them compared to opening the report via **`file:///…`**. Rewriting is intentional so video/image links match **Open** from the console. The shim **no-ops on `file:` protocol** so opening the report from disk is unchanged.
  - **Range requests:** Starlette **`FileResponse`** supports **`Range` / `206 Partial Content`** for large media when the client sends **`Range`** (not custom logic in this app).

## P1 (scaffold + Phase 1 UI)

- **Settings → Operator backups:** ZIP under an allowlisted path (default `logs/archive_console_backups`), optional `state.json` + `logs/` tree, retention by file count and optional age. **`POST /api/settings/operator-backup/run`** (blocked while a **Run** job is active). Outcomes stored in **`last_operator_backup`** in **`state.json`**.
- **Settings → Storage & retention:** Manual **Preview** / **Run cleanup** for old **`logs/archive_run_*`** folders (directory mtime) and **`archive_console_backup_*.zip`** files under the operator backup destination (file mtime). Options persist in **`state.json`** as **`storage_retention`** (default **90** days). Only paths under **archive root** that pass the **allowlist** are eligible; **`logs/latest_run*.txt` targets are never deleted**; **no cleanup while a Run job is active** (**409**). **`run_history` in `state.json` is not rewritten**—History links may 404 for pruned folders. Broken pointers are cleared after delete. **No automatic pruning on server start** in v1.
- **Settings → Scheduling:** CRUD saved to **`schedules[]`**. **Day-of-month 1–31** is **clamped** in short months (e.g. 31 → Feb 28/29). **Next-run** hints and firing use **local machine time** (optional **`TZ`** env for an IANA zone). When **`features.scheduler_enabled`** is **true** and the server is restarted, an in-process **APScheduler** tick (30s) starts the same **`.bat`** jobs as **Run** at the scheduled wall time; **overlapping** runs are skipped with a log line if a job is still **running**. **Sleep/wake:** missed ticks are **not** replayed.
- **Cookie hygiene:** optional **N-day** in-app reminder (low-priority nudge, max **14** days in UI/state; not a substitute for the Run preflight when that is enabled) + acknowledge / **short snooze** only (**1 h** / **3 h** in Settings, **15 min** on the Run banner; **server clock** is authoritative; multiple tabs are last-write-wins). Run-page + **`GET /api/settings/reminders`** include the cookie banner and the **pre-run** banner for the **earliest upcoming** enabled schedule. No browser automation; **no** logging of `cookies.txt` contents.
- **Tray pre-run balloons (optional):** when **`features.tray_notify_before_schedule`** is **true**, the console POSTs to **`http://127.0.0.1:<tray_notify_port>/notify`** during the **same** pre-run time window as the web banner (T−N minutes through kickoff, local clock). **`tray_app.py`** listens on **localhost only** and shows a **pystray** balloon. Port defaults to **`state.json` `port` + 101** when **`tray_notify_port`** is **0**; change it in **Settings** if it conflicts. If the tray app is not running, the server logs a warning and may record **`tray_notify_last_failure_*`** in **`state.json`** (see Settings).
- **`features.require_cookie_confirm_manual`:** default **`true`** — Run-tab manual starts require the cookie confirmation modal unless disabled in **Settings** or bypassed with **dry-run** (see **Runs** above).
- **`features.scheduler_enabled`:** **`true`** starts the in-process backend on server boot; **`false`** keeps schedules as saved-only (Task Scheduler / tray still useful to keep the process alive).
- **`features.notifications_stub`**: reserved for Windows toasts (**P2**).
- **Task kill / SIGTERM from other tools:** External **`taskkill`** on unrelated processes is out of scope; use **Stop run** in the UI or **`stop_server.ps1`** for the console server itself, not arbitrary yt-dlp PIDs.

## Coexistence with the rest of the repo

- **`BAT_FILES.md`** remains the **authoritative `.bat` inventory**; the console does not replace Task Scheduler or raw double-click runs.
- **`README.md`** quick-start table still describes the three **`monthly_*`** drivers; the console is an optional launcher with streaming logs.
- **Core Python drivers** (`archive_*_run.py`) are **not** forked; the console spawns the existing **`.bat`** files so **pip / backup / pointer** behavior stays aligned with interactive runs.

## Install & troubleshooting

| Issue | What to do |
|--------|------------|
| Browser opens but page / connection fails | The launcher waits with **`Start-Sleep`** via PowerShell (avoids **`ping`** / **`timeout`** quirks in some terminals). Uvicorn is started through **`archive_console/_launch_uvicorn.bat`** so **`cmd /k`** does not have to nest long quoted paths (that parsing bug can make CMD run **`python.exe` as if it were a `.py` file** → `SyntaxError` / `MZ` / `\x90`). Check the **“Archive Console (uvicorn)”** window; ensure **`http://127.0.0.1:<port>/`** matches **`state.json`** `port` (default **8756**). |
| Port already in use | Change **`port`** in **`state.json`** (or stop the other process). The launcher treats “health check succeeds” as “server already up” and only opens the browser. |
| Firewall | **N/A** for `127.0.0.1`-only binding. |
| `python` not found | Install **Python 3.10+** and ensure it is on **PATH** when running the launcher. |
| Blank logs / 409 on Run | Only **one** run at a time; wait for the current job to finish. |
| Reports 403 | Path must fall under **`allowlisted_rel_prefixes`** (default: `logs`, `playlists`, `channels`, `videos`). |
| Bookmark **`/logs`** or **`/reports`** | Redirects to **`/?view=history`** with the appropriate section anchor (same combined **History & reports** screen). |

## Tests / manual checklist

- **Automated:** from **`archive_console`** with venv active: `python -m pytest`
- **Manual P0:**
  1. Double-click **`start_archive_console.bat`** → browser opens → **Run** loads.
  2. Start a job with **dry-run** on → log stream appears; on finish, **exit code** and artifact link update. Start again → **Stop run** mid-flight → status **canceled**, log notes partial-run risk.
  3. **Files** → list only allowlisted trees; try a forged query with `..` → **403/404**.
  4. **History & reports** → **Open report** uses **`/reports/view?rel=...`** (rewritten links + inline HTML); **Download** uses **`/reports/file?…&download=1`**. From the report, a media row should target **`127.0.0.1`**, not **`file://`**. **Files** → **Open** on `.html` uses **`/reports/view`**; other types use **`/reports/file`**. Optional: open **`http://127.0.0.1:<port>/logs`** or **`/reports`** → lands on the same page with **`?view=history`**.

  5. **Download settings** → change a slider → **Save** → confirm root **`yt-dlp.conf`** updates; restart console and verify values; with a job running, **Save** should show **409**.

- **Manual tray / scheduler:**
  1. **`python tray_app.py`** (from **`archive_console`**, venv on): tray tooltip shows **Archive Console** and port; **Open web console** opens **`http://127.0.0.1:<port>/`**. If health already passes, tray **attaches** and **Exit** leaves the server running. A small **HTTP** listener on **`127.0.0.1:<tray_notify_port>`** accepts **`POST /notify`** (JSON **`title`**, **`body`**) for optional pre-run cookie reminders when enabled in Settings.
  2. Spawn mode: **Exit** stops uvicorn; with a **Run** job active, confirm the force dialog (Windows) or policy on other OS.
  3. Set **`features.scheduler_enabled`:** **`true`** in **`state.json`**, restart → Settings reports an active backend; at a test schedule time, verify **`scheduler: starting job`** in logs and a history entry.

- **Manual cookie + tray checklist:**
  1. With **`require_cookie_confirm_manual`** on (default), start a real run without the checkbox modal → first **`POST /api/run/start`** returns **428**; **Continue** with **I’ve updated cookies.txt** sends **`cookie_confirm: true`** and starts the job.
  2. **Cancel** in the modal leaves the job idle; **dry-run** skips the gate without the modal.
  3. Enable **`tray_notify_before_schedule`**, set **`pre_run_reminder.minutes_before`** & an upcoming enabled schedule, run **`tray_app.py`**, and confirm one balloon in the pre-run window (no spam across 30s ticks). Stop the tray and verify the server logs / **last tray notify** line in Settings on failure paths.

### Tray icon (assets)

- **Shipped files:** **`archive_console/assets/tray.ico`** (embedded **16, 24, 32, 48**, plus **256** for sharp rescale) and PNG sources **`assets/tray.png`** (256) and **`assets/tray_64.png`**. Motif: **bold gold arrow-into-folder** on a matte black tile. The tray loads the **largest** ICO frame for pystray, which re-serializes for the shell.
- **Regenerate (maintainers):** from **`archive_console`**, run **`python assets/build_tray_icon.py`**. Raster logic lives in **`tray_icon_raster.py`** (shared with the in-process **fallback** if **`tray.ico`** is missing).
- **Dev vs frozen path:** runtime resolves **`archive_console/assets/tray.ico`**. Under **PyInstaller** (Windows), bundle the ICO next to the extract root as **`assets/tray.ico`**, for example from **`archive_console`**:  
  **`pyinstaller --noconsole --onefile --hidden-import tray_icon_raster --add-data "assets/tray.ico;assets" tray_app.py`**  
  (Use **`;`** as the **`--add-data`** separator on Windows; on Linux/macOS use **`:`.**)
- **Missing file:** the app logs a **warning** and draws a **generated** black/gold chip (same geometry as the builder) so behavior never silently falls back to an unrelated default.
- **HiDPI:** **`tray_app`** attempts **per-monitor DPI awareness** on Windows before creating the icon (best effort). If the icon looks soft at **125% / 150%**, confirm the process is not **system DPI–unaware** (Python / frozen manifest) and that **`tray.ico`** contains crisp **16–32** entries.
- **Stale icon in the shell (rare):** Windows may cache notification-area images. If an old tray icon persists after an update, try **sign out/in**, restarting **Explorer**, or (advanced): **`ie4uinit.exe -show`** — see Microsoft / community notes on notification icon refresh.
