      Settings
      
          
      Runtime, backups, schedule records, and cookie reminders. Schedules only run while the server process stays up (use the tray launcher or a Windows service — not a closed CMD window).
      
        
      

        
      
          
      General & runtime
      
          
      Binds to 127.0.0.1 only (not LAN). Changing the port requires restarting the console process.
      
          
      
          
          
          
          
          
      
        
      

        
      
          
      Storage & retention
      
          
      Manual cleanup only—nothing deletes on server start. Eligible paths must sit under the archive root and allowlist. Run folders use directory mtime; operator ZIPs use file mtime. logs/latest_run*.txt targets are always kept (even if older than retention). A job must not be running. cookies.txt, yt-dlp.conf, and console state.json are never deleted. History may reference removed folders—the report link can 404 until you run again. Editor .bak files live under archive_console/backups (outside the archive tree) and are not pruned here.
      
          
      Not affiliated with yt-dlp.
      
          
          
          
          
      
            
            
            
          
      
          
      
          
      
        
      

        
      
          
      Headless / tray (Windows)
      
          
      Use start_archive_console_tray.bat (repo root) or python tray_app.py from archive_console with venv + pystray. This calls the same uvicorn argv as CLI (app.server_cli.uvicorn_argv). If the port is already serving /api/health, the tray attaches and Exit closes only the tray. Otherwise it spawns the server and Exit stops it (blocks if a job is running unless you confirm). See ARCHIVE_CONSOLE.md.
      
        
      

        
      
          
      Operator backups (ZIP)
      
          
      Creates archive_console_backup_*.zip under a path you choose (must match an allowlisted prefix, e.g. logs/…). Includes optional copies of console state.json and logs/. Does not read cookies.txt unless you add that path under allowlist and include it manually (not recommended in shared backups).
      
          
          
          
          
          
          
          
      
            
            
          
      
          
      
          
      
        
      

        
      
          
      Scheduling (saved entries)
      
          
      
          
      Day of month 1–31: in short months the day is clamped to the last valid day (e.g. 31 → Feb 28/29). Next-run hints and the in-process scheduler use this machine’s local timezone. Enable features.scheduler_enabled in state.json and restart the server to run jobs here; missed minutes while the PC sleeps are not replayed.
      
          
      
          
          
          
      
        
      

        
      
          
      Cookie hygiene (reminders)
      
          
      In-app reminders only. No browser automation or silent export. Use Inputs & config → cookies.txt after you export from the browser yourself.
